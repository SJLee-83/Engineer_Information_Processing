# OAuth (Open Authorization)

인터넷 애플리케이션에서 사용자 인증에 사용되는 표준 인증 방법으로, 공개 API로 구현되었다."

    OAuth는 공개 API(Public API) 환경에서 인증 및 인가를 위한 표준 프로토콜입니다.

"인터넷 사용자가 웹사이트나 애플리케이션에 비밀번호를 제공하지 않고 자신에게 접근 권한을 부여하여 사용할 수 있다."

    이것이 OAuth의 가장 핵심적인 기능인 **위임된 인가(Delegated Authorization)**입니다.

    예를 들어, 'OOO앱'이 "구글 계정으로 로그인" 기능을 제공할 때, 사용자는 OOO앱에 자신의 구글 비밀번호를 알려주지 않습니다. 
    대신 구글에 로그인하여 "OOO앱이 내 프로필 정보를 볼 수 있도록 권한을 허용하시겠습니까?"라는 창에 동의합니다. 
    이 과정에서 OOO앱은 비밀번호 대신 '액세스 토큰(Access Token)'을 받아 구글의 자원에 접근하게 됩니다.

"2010년 IETF에서 1.0이 공식 표준안으로 발표되었다."

    OAuth 1.0a (OAuth 1.0의 개정판)가 2010년에 IETF RFC 5849로 발표되었습니다. (이후 2012년에 OAuth 2.0이 표준화되어 현재 널리 사용되고 있습니다.)

# OpenID (Open Identifier)

    인증(Authentication) 프로토콜입니다. "당신이 누구인지(Who you are)"를 증명하는 데 중점을 둡니다.

하나의 ID와 비밀번호로 여러 사이트에 로그인할 수 있게 해주는 개념입니다. OAuth와 자주 혼동되지만, OpenID는 신원 확인이 목적입니다.

# IDEA (International Data Encryption Algorithm)

    암호화 알고리즘입니다. AES, DES와 같은 대칭키 블록 암호 방식 중 하나로, 데이터를 암호화하고 복호화하는 기술입니다. 문제의 인증/인가 프로토콜과는 관련이 없습니다.

# SSPI (Security Support Provider Interface)

    마이크로소프트 윈도우 운영체제에서 사용하는 보안 API입니다. Kerberos나 NTLM 같은 다양한 보안 인증 공급자(Provider)를 통합적으로 사용할 수 있게 해주는 인터페이스입니다. 
    인터넷 표준 프로토콜이라기보다는 윈도우 시스템 내부의 개념입니다.

# SASL (Simple Authentication and Security Layer)

    SMTP(메일), LDAP(디렉터리) 같은 인터넷 프로토콜에서 인증 및 보안을 제공하기 위한 프레임워크입니다. 특정 인증 메커니즘을 프로토콜에 쉽게 '플러그인'할 수 있도록 해줍니다.

# PEAP (Protected Extensible Authentication Protocol)

    주로 **무선 네트워크(Wi-Fi)**의 WPA-Enterprise 환경에서 사용되는 보안 인증 프로토콜입니다.

# OIDC (OpenID Connect)

OAuth 2.0 위에 구축된 인증 계층입니다.

    즉, OAuth 2.0의 '인가' 기능(토큰 발급)을 기반으로, OpenID의 '인증' 기능(사용자 신원 정보 제공)을 합친 최신 표준입니다. 
    "Log in with Google"은 실제로는 OAuth 2.0과 OIDC를 함께 사용하는 경우가 많습니다.

# JAAS (Java Authentication and Authorization Service)

    자바(Java) 플랫폼에서 사용자를 인증하고 접근 제어를 수행하기 위한 API 및 프레임워크입니다. 자바 애플리케이션을 위한 보안 표준입니다.
